## 1. 这是什么软件

**轻记账** 是一款 **个人记账软件** 的 **电脑端先行版本**。

### 1.1 产品定位

| 维度 | 说明 |
|------|------|
| **目标用户** | 个人 / 家庭记账 |
| **使用场景** | 日常收支记录、账户管理、统计分析、预算控制 |
| **数据策略** | **本地优先**，所有账单数据保存在本机，不上传云端 |
| **双端规划** | 当前只做电脑端；核心层代码预留手机端复用；后续支持局域网同步到手机 |
| **当前版本** | 0.1.0，功能已基本覆盖记账全流程 |

### 1.2 核心能力摘要

- 支出 / 收入 / 转账 三类交易
- 多账本、多账户、多级分类、标签
- 首页 Dashboard、统计图表、日历/列表/相册三种交易视图
- 预算管理、定时交易、借贷记录
- 账单导入（CSV / Excel / 支付宝账单）、导出（CSV / Excel / SQLite）
- 本机 OCR 识图记账（微信/支付宝截图等）
- 实时汇率查询（160+ 法币，每日更新，本地缓存）
- 系统托盘、全局快捷键、主题切换
- 同步中心（UI 与框架已预留，尚未完整打通手机端）

## 2. 架构说明：有没有前后端

### 2.1 重要结论

**本项目不是传统的「前端 + 后端服务器」架构。**

它是一个 **单机桌面应用（Client-Only）**：

```
┌─────────────────────────────────────────────────────────┐
│                     轻记账 Desktop                      │
│  ┌──────────────────┐    ┌──────────────────────────┐ │
│  │   UI 层 (desktop) │───▶│  业务层 (core/services)  │ │
│  │  Flutter Widgets  │    │  记账 / 统计 / 导入导出   │ │
│  │  Riverpod 状态    │    │  OCR 解析 / 汇率 / 备份   │ │
│  └──────────────────┘    └───────────┬──────────────┘ │
│                                       │                  │
│                          ┌────────────▼──────────────┐  │
│                          │   数据层 (core/database)   │  │
│                          │   DAO + SQLite (本地文件)  │  │
│                          └───────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
          │ 可选联网（仅汇率等辅助功能，非核心账单）
          ▼
   外部公开 API（如 jsDelivr 汇率 CDN）
```

### 2.2 分层对照（便于理解）

| 常见叫法 | 在本项目中的对应 | 技术 |
|----------|------------------|------|
| **前端 / UI** | `lib/desktop/` | Flutter + Material 3 风格主题 |
| **后端 / 业务逻辑** | `lib/core/services/` | Dart 纯业务服务 |
| **数据库** | `lib/core/database/` | SQLite（sqflite_common_ffi） |
| **后端服务器** | **无** | 不需要 Node/Java/Go 等服务端 |
| **API 接口** | **无 REST API** | 页面通过 Riverpod 直接调 Service / DAO |

### 2.3 为什么要分 core 和 desktop

- **`core/`**：与平台无关的核心逻辑，**后续可抽离为共享包**，供手机端复用
- **`desktop/`**：仅桌面端特有的 UI、窗口、托盘、快捷键等

这是 **「一套核心 + 多端 UI」** 的架构，而不是前后端分离。

| 表名 | 说明 |

| `budgets` | 预算 |
| `scheduled_transactions` | 定时/周期交易 |
| `loans` | 借贷记录 |
| `sync_records` | 同步记录（预留） |

### 5.3 核心模型（`lib/core/models/`）

| 模型 | 说明 |
|------|------|
| `Book` | 账本 |
| `Account` | 账户 |

| 枚举 | 值 |

## 10. 外部数据与 AI 能力

### 10.1 本机 OCR（账单识图）

| 项目 | 内容 |
|------|------|
| 插件 | `mobile_ocr` |
| 模型 | PaddleOCR **PP-OCRv5**（`pp-ocrv5-202410`） |
| 体积 | 约 **21 MB**（det 4.6M + rec 16M + cls 0.57M + 字典） |
| 推理 | ONNX Runtime（`onnxruntime_v2`） |
| 网络 | **不需要**，完全离线 |
| 后续 | 本地规则解析 → 填单，非云端大模型 |

### 10.2 实时汇率

| 项目 | 内容 |
|------|------|
| 数据源 | fawazahmed0/exchange-api（jsDelivr CDN） |
| API Key | 不需要 |
| 更新 | 每日 |
| 本地缓存 | `Documents/ezbookkeeping/exchange_rates/` |

### 10.3 同步（未来）

- 协议与 HTTP 服务框架预留
- 依赖 `dio` + `multicast_dns`
- 当前 **不会上传账单到任何云端**
