## 1. 项目定位与架构

### 1.1 这是什么

**轻记账（ezBookkeeping Desktop）** 是一款 **个人记账软件** 的电脑端先行版本：

| 维度 | 说明 |
|------|------|
| 数据策略 | **本地优先**，SQLite 存于用户文档目录，默认不上传云端 |
| 双端规划 | 当前只做桌面端；`core/` 层预留手机端复用 |
| 同步规划 | 后续局域网与手机端协同（框架已预留，协议未打通） |
| 架构形态 | **单机客户端**，无独立后端服务器 |

### 1.2 技术栈（强制）

| 层级 | 技术 |
|------|------|
| 框架 | Flutter Desktop (Dart) |
| 数据库 | SQLite（sqflite_common_ffi） |
| 状态管理 | Riverpod |
| 路由 | go_router |
| 图表 | fl_chart |
| 网络 | dio（预留同步） |
| 设备发现 | multicast_dns（预留） |
| 桌面能力 | window_manager / system_tray / hotkey_manager / file_picker |
| OCR | mobile_ocr（PaddleOCR PP-OCRv5，本机离线） |

### 1.3 分层结构

```
lib/
├── core/          # 平台无关：模型、DAO、业务服务、AI 规则引擎
└── desktop/       # 桌面端 UI、Provider、窗口/托盘/快捷键
```

**原则**：UI 不直接操作数据库；所有读写经 DAO / Service；`core/` 不引入桌面专属包。

## 2. 开发阶段总览

根据 `.cursorrules` 规划的 6 个阶段，当前整体进度如下：

| 阶段 | 规划内容 | 实际状态 |
|------|----------|----------|
| **Phase 1** | 项目初始化、数据库、模型、DAO、Sidebar、页面骨架 | ✅ **已完成** |
| **Phase 2** | 首页 Dashboard、记一笔、交易列表 | ✅ **已完成** |
| **Phase 3** | 日历、统计、预算展示 | ✅ **已完成** |
| **Phase 4** | 账户/账本/分类/借贷/周期记账 | ⚠️ **大部分完成**（周期记账已移除） |
| **Phase 5** | 设置、导入导出、托盘、快捷键、通知 | ⚠️ **大部分完成**（自动通知/自动备份未真正调度） |
| **Phase 6** | 同步中心 UI、mDNS/HTTP 框架 | ⚠️ **仅 UI + 空壳服务** |

**结论**：桌面端 **记账主流程已可日常使用**；同步、周期记账、部分系统能力仍为占位或待完善。

## 4. 部分完成 / 已知问题

### 4.1 功能「有 UI 无实现」

| 项目 | 现状 |
|------|------|
| **每日记账提醒** | 设置里可开关、选时间，但 **未接入 `local_notifier`**，到点不会真正弹通知 |
| **自动备份** | 可配置周期（日/周/月）和保留天数，但 **无后台定时任务** 自动执行备份 |
| **同步中心** | 页面完整，「发现设备」返回空列表，「开始同步」按钮禁用 |
| **mDNS 服务** | `MdnsService.startDiscovery()` 直接 `UnimplementedError` |
| **数据加密** | `EncryptUtils` 仅占位，抛 `UnimplementedError` |
| **AI 分类器接口** | `CategoryClassifier` 声明了 fastText/大模型接口，**未实现**；当前靠规则匹配 |
| **通用组件占位** | `AppDateRangePicker`、`AppDataTable`、`ChartCard` 仍为 `Placeholder()`（实际页面用了专用实现） |

### 4.2 数据与业务已知问题

| 问题 | 说明 | 缓解措施 |
|------|------|----------|
| **历史导入错账** | 早期导入可能把支出挂到「现金」 | 账户页 → 修复账户余额（需 comment 含 `pay=` 元数据） |
| **无元数据旧账单** | 很早导入、comment 无 `@src:` 的账单无法自动修复归属 | 需后续「批量重映射」或重新导入 |
| **花呗账户类型显示「无」** | 部分账户 type 为 `none` | 启动时 `ensureDefaultAccounts` 会尝试修正预设账户类型 |
| **搜索页月份统计** | 基于**已加载**列表计算，未滚到底时可能不完整 | 属性能取舍，可后续改为 DB 聚合 |
| **模型未用 freezed** | 规范要求 freezed，实际为手写模型 | 不影响运行，后续可逐步迁移 |

### 4.3 已移除或变更的规划

| 原规划 | 当前状态 |
|--------|----------|
| **周期记账 / 定时交易** | 数据库 v7 **已删除** `scheduled_transactions` 表；无独立页面与服务 |
| **侧边栏「记一笔」** | 已移除；记账入口改由总览/交易详情/托盘/快捷键 |
| **侧边栏「定时交易」** | 从未在现行 sidebar 中上线 |
